Expand description

Defines a “server function.” A server function can be called from the server or the client, but the body of its code will only be run on the server, i.e., if a crate feature ssr is enabled.

Server functions are created using the server macro.

The set of server functions can be queried on the server for routing purposes by calling [server_fn::ServerFunctionRegistry::get].

Technically, the trait is implemented on a type that describes the server function’s arguments, not the function itself.
